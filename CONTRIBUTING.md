# Contributing

This framework gets better when its claims are tested and its reach is widened. Four kinds of contribution are welcome.

1. **Corrections.** A misattributed citation, a factual error, a broken link, a claim that overreaches the evidence. Open an issue or a pull request with the source.
2. **Challenges.** A well-argued disagreement with a driver, a classification, or a boundary claim. The framework states in Part VI where it can be proven wrong; if you can show it is wrong somewhere, that is a contribution, not an attack. Bring the evidence.
3. **Translations.** The framework lives in English and Indonesian, kept in step. Other languages are welcome as parallel files (`wtp-framework.<lang>.md`). A translation should match the structure of the canonical English file section for section.
4. **Worked examples.** A real application of the framework, run through the diagnostic method, added to `examples/`. Strip anything confidential.

## How to propose a change

- Small fixes: open a pull request directly.
- Larger changes (a new driver, a restructured layer, a new context): open an issue first so the reasoning can be discussed before the work.
- Keep the two language files in step. A change to the English framework needs the matching change in the Indonesian one, and the reverse.

## Writing standard for prose

The framework is written to read like a person wrote it, not a model. If you contribute prose, please follow the same standard:

- No em dashes. Use commas, colons, parentheses, or a new sentence.
- Avoid contrast-framing as a device ("not X but Y", "bukan X melainkan Y"). State the point directly.
- Avoid filler and inflated vocabulary. Plain, precise words.
- Cite sources for factual claims. Do not invent statistics or studies.

# Berkontribusi (Bahasa Indonesia)

Kerangka ini membaik ketika klaimnya diuji dan jangkauannya diperluas. Empat jenis kontribusi diterima.

1. **Koreksi.** Sitasi yang salah atribusi, kesalahan fakta, tautan rusak, klaim yang melebihi bukti. Buka issue atau pull request dengan sumbernya.
2. **Bantahan.** Ketidaksetujuan yang berargumen baik terhadap sebuah pendorong, klasifikasi, atau klaim batas. Bagian VI menyatakan di mana kerangka bisa dibuktikan salah; kalau Anda bisa menunjukkannya salah di suatu tempat, itu kontribusi. Bawa buktinya.
3. **Terjemahan.** Kerangka hidup dalam Bahasa Inggris dan Indonesia yang dijaga setara. Bahasa lain diterima sebagai file paralel (`wtp-framework.<lang>.md`), mengikuti struktur file kanonik bagian demi bagian.
4. **Contoh terapan.** Penerapan nyata yang dijalankan lewat metode diagnostik, ditambahkan ke `examples/`. Hapus apa pun yang rahasia.

## Cara mengusulkan perubahan

- Perbaikan kecil: buka pull request langsung.
- Perubahan besar (pendorong baru, lapis yang ditata ulang, konteks baru): buka issue dulu agar alasannya bisa didiskusikan sebelum dikerjakan.
- Jaga kedua file bahasa tetap setara. Perubahan pada file Inggris menuntut perubahan padanan pada file Indonesia, dan sebaliknya.

## Standar penulisan prosa

Kerangka ditulis agar terbaca seperti tulisan manusia. Kalau Anda menyumbang prosa, ikuti standar yang sama: tanpa em dash, hindari pola kontras sebagai gaya, hindari kata pengisi dan kosakata yang dibesar-besarkan, dan sitasi sumber untuk klaim faktual.

## Lisensi kontribusi

Dengan berkontribusi, Anda setuju kontribusi Anda mengikuti lisensi proyek: kontribusi untuk kode tool memakai Lisensi MIT, kontribusi untuk teks framework memakai Creative Commons Attribution 4.0 International (CC BY 4.0).
